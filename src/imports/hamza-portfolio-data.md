### PROJECT 1 — VERTEX IDS (INTELLIGENT DATA SYSTEM)
**Type:** Machine Learning / NLP / iOS Deployment
**Status:** Completed
**Collaborator:** M. Shahzaib (01-14232-132)

**Pitch:**
End-to-end ML pipeline that classifies Software Requirement Specifications (SRS) into Functional Requirements, Non-Functional Requirements, and Outlier categories — deployed as a fully functional iOS application.

**Problem:**
Manual classification of software requirements is time-consuming, inconsistent, and error-prone. SRS documents contain hundreds of requirement statements with no automated categorization layer.

**Solution:**
Engineered a complete NLP-driven classification system capable of processing raw SRS text and outputting structured category labels with high accuracy — wrapped in a cross-platform mobile interface.

**ML Pipeline:**
- Text preprocessing & cleaning
- Tokenization & feature extraction
- Synthetic dataset augmentation
- Multi-class classification model training
- Accuracy, Precision, Recall, F1 evaluation
- High classification accuracy score achieved

**Engineering Stack:**
- Python (ML/NLP core)
- Scikit-learn / TensorFlow
- NLP preprocessing pipeline
- FastAPI / Flask backend
- iOS deployment (CoreML or API bridge)
- Flutter mobile layer
- Firebase / MySQL (data persistence)

**Architecture:**
`Raw SRS Text → Preprocessing → Feature Extraction → Trained Classifier → API Layer → iOS App Interface`

**Differentiators:**
- Synthetic data augmentation to handle class imbalance
- Full end-to-end execution: research → model → deployed app
- iOS app shipped — not just a Jupyter notebook
- Demonstrates NLP + mobile deployment in a single pipeline

**Challenges Overcome:**
- Dataset imbalance across requirement categories
- Text ambiguity in real-world SRS inputs
- Bridging ML model inference to mobile runtime
- Integration of backend API with iOS application layer

**What to Show:**
